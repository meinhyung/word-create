# 단어 승부 (word-duel)

커플/친구와 겨루는 실시간 글자 대결 게임. 각자 글자를 하나씩 고르면 공개되고,
그 두 글자를 모두 포함한 단어를 먼저 완성하는 사람이 라운드를 가져간다. 3선.

## 규칙

1. 라운드가 시작되면 제한시간(EASY/MEDIUM 5초, HARD 7초) 안에 각자 글자를 하나씩 고른다.
   시간 안에 고르지 않으면 랜덤 글자가 배정된다.
2. 두 글자가 공개되고, 두 글자를 모두 포함한 단어를 먼저 제출하는 사람이 승리.
   - **EASY**: 시간 제한 없음. 두 글자가 순서 상관없이 포함되면 인정
   - **MEDIUM**: 15초 제한. 그 라운드의 "선(先)" 플레이어 글자가 단어에 먼저 나와야 함
   - **HARD**: 10초 제한. MEDIUM과 동일한 순서 규칙 + 단어는 5글자 이상이어야 함
   - 두 사람이 같은 글자를 고르면, 그 글자가 단어에 두 번 포함되어야 인정
3. 라운드를 이긴 사람이 다음 라운드의 "선"이 된다. 시간 안에 아무도 완성 못하면 무승부 처리되고
   다음 라운드의 선은 랜덤으로 재배정된다 (EASY는 시간 제한이 없어 무승부가 나지 않는다).
4. 먼저 3점을 딴 사람이 매치 승리. 화면에는 3칸짜리 점(pip) 그래프로 스코어가 표시된다.

## 단어 검증

알파벳(영어)만 지원한다. `nspell` + `dictionary-en`(허니spell 기반 오픈소스 영어 사전)으로 실제
단어인지 서버가 자동 검증한다. (한글 자음 모드도 시도했었지만, 오픈소스 한글 사전이 사실상 모든
글자 조합을 통과시켜버려 신뢰할 수 없었고 게임 난이도도 너무 어렵다는 피드백으로 제외했다.)

## 방 참가 방식

- **초대 코드**: 방을 만드는 사람이 난도를 직접 정하고, 코드를 공유받은 상대는 그대로 참가한다.
- **랜덤 매칭**: 대기열에서 자동으로 상대와 연결되며, 이때는 난도가 서버에 의해 무작위로 정해진다.

## 로컬 실행

```bash
npm install
npm start
```

`http://localhost:3001` 접속. 혼자 테스트하려면 탭 2개를 열면 된다.

시간 설정을 테스트용으로 빠르게 돌리고 싶으면:

```bash
TIME_SCALE=0.3 npm start
```

## 구조

| 파일 | 역할 |
|---|---|
| `server.js` | 방/매치 상태, 글자·단어 판정, 사전 검증(nspell), 라운드/스코어 진행 (Socket.IO) |
| `public/index.html` | 게임 화면 전체 (HTML + CSS + JS), 단어 텔레파시와 동일한 디자인 톤 |

## 설계 시 확정한 사항 (참고)

- 글자 선택 제한시간(EASY/MEDIUM 5초, HARD 7초)과 단어 제출 제한시간(EASY 무제한/MEDIUM 15초/HARD 10초)은
  서로 다른 값으로 독립적으로 운영된다.
- 라운드 결과 후 다음 라운드까지 약 2.5초 연출 대기, 매치 시작 전 약 2초 대기
- 매치 종료 후 "재대결" 투표는 단어 텔레파시와 동일한 방식(둘 다 동의해야 재시작, 한 명이라도 거절하면 종료)
